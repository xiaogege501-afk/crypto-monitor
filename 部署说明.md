# 网页版部署说明 —— 打开浏览器就能用

这是桌面工具的网页版：把它放到一台服务器上跑起来后，你（以及你允许的任何人）
只要在浏览器里输入 `http://服务器地址:5000`，就能看到持仓和推荐币种的实时多周期趋势，
手机、平板、电脑都能打开，不需要在每台设备上装 Python。

界面和功能与桌面版一致：持仓管理、强势推荐扫描、15分钟/1小时/日线/周线四周期共振提示。

---

## 需要准备什么

一台 **有公网 IP 的服务器**（VPS）。国内的阿里云/腾讯云轻量服务器也可以，
但因为数据来自币安接口，国内服务器直连币安大概率会被墙，需要在设置里配置代理
（和桌面版一样，界面下方「代理地址」那一栏）。**如果条件允许，更推荐用境外服务器**
（如新加坡、日本节点的 Vultr、DigitalOcean、Linode 等），这样不需要代理就能直接访问币安接口，速度也更快。

服务器操作系统建议 Ubuntu 22.04 / 20.04，配置很低即可（1核1G足够）。

---

## 方式一：Docker 部署（最省心，推荐）

1. 把 `crypto_web` 整个文件夹上传到服务器（用 `scp`、`rsync`，或宝塔面板的文件管理都行）
2. SSH 登录服务器，安装 Docker（如果还没装）：
   ```bash
   curl -fsSL https://get.docker.com | bash
   ```
3. 进入项目目录，一键启动：
   ```bash
   cd crypto_web
   docker compose up -d --build
   ```
4. 打开浏览器访问：`http://你的服务器IP:5000`
5. 以后想停止/重启：
   ```bash
   docker compose down      # 停止
   docker compose up -d     # 启动
   docker compose logs -f   # 查看日志
   ```

配置文件 `config.json`（持仓列表、代理、刷新间隔）会持久化保存在服务器上的同名文件里，
重启容器不会丢失。

---

## 方式二：不用 Docker，直接用 Python 运行

1. 服务器上安装 Python 3.9+：
   ```bash
   sudo apt update && sudo apt install -y python3 python3-pip
   ```
2. 上传 `crypto_web` 文件夹后，安装依赖：
   ```bash
   cd crypto_web
   pip3 install -r requirements.txt
   ```
3. 用 gunicorn 以生产模式启动（比直接 `python app.py` 更稳定）：
   ```bash
   gunicorn -w 2 -b 0.0.0.0:5000 --timeout 60 app:app
   ```
4. 想让它在后台一直运行、且服务器重启后自动拉起，建议配成 systemd 服务：

   创建 `/etc/systemd/system/crypto-monitor.service`：
   ```ini
   [Unit]
   Description=Crypto Monitor Web
   After=network.target

   [Service]
   WorkingDirectory=/root/crypto_web
   ExecStart=/usr/bin/gunicorn -w 2 -b 0.0.0.0:5000 --timeout 60 app:app
   Restart=always
   User=root

   [Install]
   WantedBy=multi-user.target
   ```
   然后：
   ```bash
   systemctl daemon-reload
   systemctl enable --now crypto-monitor
   systemctl status crypto-monitor
   ```

## 打开防火墙端口

如果访问不了，先检查服务器安全组/防火墙有没有放行 5000 端口：

```bash
# ufw 示例
sudo ufw allow 5000/tcp
```

云服务商的控制台（阿里云"安全组"、腾讯云"防火墙"等）也要单独放行 5000 端口的入站规则。

## 想用域名 + https 访问（可选，进阶）

如果你有自己的域名，可以用 Nginx 做反向代理，把 80/443 转发到内网的 5000 端口，
再用 certbot 免费申请 SSL 证书，这样就能用 `https://你的域名` 访问，也更安全。这一步不是必须的，
纯 IP:端口 也完全能正常使用。需要的话可以告诉我，我再补一份 Nginx + certbot 的详细配置。

## 关于安全性和隐私

这个网页目前**没有登录验证**，只要知道你的服务器 IP 和端口，任何人都能打开看到你的持仓列表
（不涉及私钥、API Key、资金，只是一个观察列表，但如果你不想被外人看到，建议：

- 只把端口开放给你自己常用的 IP（云服务商安全组里可以按来源 IP 限制）
- 或者用 Nginx 加一个简单的 Basic Auth 账号密码
- 或者只在服务器本地运行（不开放公网端口），通过 SSH 隧道 `ssh -L 5000:localhost:5000 user@服务器IP`
  在你自己电脑上访问 `http://localhost:5000`，这样外部完全访问不到

如果需要，我可以帮你加一个简单的登录密码保护。

## 常见问题

**Q: 页面打开了，但持仓/推荐一直转圈或报错？**
A: 大概率是服务器访问不了币安接口。国内服务器需要在「设置」里填代理地址；
境外服务器一般不需要，可以先用 `curl https://api.binance.com/api/v3/ping` 在服务器上测试网络连通性。

**Q: 扫描推荐要等很久？**
A: 正常情况下 10-30 秒左右，因为需要拉取几十个币种的行情再筛选，属于正常耗时，
页面上会显示扫描进度文字。

**Q: 我想让好几个人都能看，但配置各自的持仓怎么办？**
A: 当前版本是单人单持仓列表设计（所有访问者看到同一份持仓）。如果需要多用户各自独立的持仓，
需要加登录和数据库支持，告诉我需求我可以再扩展。
